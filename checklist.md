# Checklist：竞品分析 Design Skill

## 一、问题定义
- [ ] 是否明确了本次竞品分析要解决的问题？
- [ ] 是否明确问题属于业务层、框架层还是功能层？
- [ ] 是否说明了分析结果最终要支持什么决策？
- [ ] 是否定义了分析边界，避免范围失控？

## 二、竞品选择
- [ ] 是否解释了每个竞品的入选原因？
- [ ] 竞品是否与当前问题高度相关？
- [ ] 是否兼顾头部样本、直接对手或专项优秀样本？
- [ ] 是否避免为了凑数量而盲目扩样？

## 三、体验目标与维度拆解
- [ ] 是否先识别了竞品方案的体验目标？
- [ ] 是否明确竞品优先优化的是效率、理解、信任、转化还是探索？
- [ ] 分析维度是否由问题导出，而不是机械套模板？
- [ ] 是否覆盖关键信息组织、流程、决策支持等核心维度？

## 四、内容解构
- [ ] 是否不仅记录“做了什么”，还解释了“为什么这样设计”？
- [ ] 是否识别了各竞品在关键维度上的不同做法？
- [ ] 是否避免停留在截图拼盘层面？
- [ ] 是否将竞品分析聚焦在设计决策相关信息上？

## 五、方案归纳
- [ ] 是否从具体产品做法中归纳出典型方案类型？
- [ ] 是否说明了每类方案的适用条件？
- [ ] 是否说明了每类方案的优势？
- [ ] 是否识别了代价、风险与限制？

## 六、自身位置判断
- [ ] 是否将自身产品放入同一坐标系进行比较？
- [ ] 是否明确自身当前采用的是哪类方案？
- [ ] 是否判断了自身处于保守、主流还是领先位置？
- [ ] 是否识别了差距来源与差异化机会？

## 七、策略输出
- [ ] 是否输出了清晰的设计策略，而不是只做现象总结？
- [ ] 是否说明了建议采用什么方向？
- [ ] 是否说明了不建议采用什么方向？
- [ ] 是否结合了业务现状与限制条件？
- [ ] 是否提出了后续需要验证的关键假设？

## 八、方法论质量
- [ ] 输出是否足够结构化？
- [ ] 输出是否可复用，而不只服务于单次项目？
- [ ] 输出是否支持团队达成共识？
- [ ] 输出是否真正支撑了设计决策？

## 九、最终判定
如果以下四项同时满足，则这份竞品分析基本合格：
- [ ] 问题清晰
- [ ] 方案归纳成立
- [ ] 自身位置判断明确
- [ ] 有策略输出
