# Example Output：期望输出示例

## 一句话总结
竞品分析的目标，不是罗列别人怎么做，而是围绕明确问题识别行业解法、归纳典型方案、判断自身位置，并形成设计策略。

## 核心框架
1. 明确问题定义
2. 筛选相关竞品
3. 识别体验目标
4. 拆解分析维度
5. 归纳典型方案类型
6. 判断自身位置
7. 输出设计策略

## 示例准则
- 先定义问题，再开始竞品分析
- 竞品选择必须服务于问题
- 先识别体验目标，再拆维度
- 从具体做法中归纳方案类型
- 将自身产品纳入同一坐标系比较
- 竞品分析必须输出策略，而不止是结论

## 适合作为 Design Skill 的输出形式
- 方法论文档
- 竞品分析框架
- 模板
- 评审清单
- Prompt
